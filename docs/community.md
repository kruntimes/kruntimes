# Community and Project Governance

This page collects the project documents that define how people participate,
get help, and report issues.

## Maintainers

Current maintainers are listed in
[MAINTAINERS.md](https://github.com/kruntimes/kruntimes/blob/main/MAINTAINERS.md).

Project decision-making and maintainer changes are described in
[GOVERNANCE.md](https://github.com/kruntimes/kruntimes/blob/main/GOVERNANCE.md).

## Contributing

Before opening a PR, read
[CONTRIBUTING.md](https://github.com/kruntimes/kruntimes/blob/main/CONTRIBUTING.md).

Contributors are expected to follow the
[Code of Conduct](https://github.com/kruntimes/kruntimes/blob/main/CODE_OF_CONDUCT.md).

## Support

Usage questions, bug reports, feature requests, and support expectations are
described in
[SUPPORT.md](https://github.com/kruntimes/kruntimes/blob/main/SUPPORT.md).

## Security

Do not report vulnerabilities through public issues. Use the private reporting
channel described in
[SECURITY.md](https://github.com/kruntimes/kruntimes/blob/main/SECURITY.md).

The technical trust boundary is documented in
[Security and Threat Model](security.md).
