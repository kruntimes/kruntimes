# Governance

kruntimes uses a maintainer-led governance model while the project and its
`v1alpha1` APIs are still experimental.

## Roles

Contributors participate through issues, reviews, documentation, code, testing,
and other project work.

Maintainers are responsible for:

- reviewing and merging changes;
- defining release scope and compatibility policy;
- responding to security and conduct reports;
- administering project infrastructure;
- keeping project decisions and roadmaps visible.

Current maintainers are listed in [MAINTAINERS.md](MAINTAINERS.md).

## Decisions

Routine decisions are made through issue and pull request review. Maintainers
seek technical consensus based on correctness, operational impact,
maintainability, and alignment with the project scope.

Changes to CRD or gRPC APIs, execution semantics, security boundaries,
installation scope, or governance require an issue or design document before
implementation. If consensus cannot be reached, the maintainers make the final
decision and document the reasoning.

## Maintainer Changes

New maintainers are selected from contributors who demonstrate sustained
technical judgment, constructive review, reliability, and care for the broader
project. Existing maintainers approve additions and removals by consensus.

Maintainers may step down at any time. A maintainer may be removed for prolonged
inactivity, repeated violations of the Code of Conduct, or actions that put the
project or its users at material risk.

## Amendments

Changes to this document require maintainer approval through a pull request.
